Parts animation script for Space Engineers
Exporting in Blender

Installing

First of all you need to install the Python script in Blender.

  1. Open Blender
  2. Go to Files -> User preferences...
  3. Click "Install from File..." at the bottom
  4. Find "Anima_Blender.zip" and select it
  5. Under "Space Engineers" category find "Anima Script Export"
  6. Click the tiny checkbox on the right to load
  7. Click "Save User Settings" to save the configurations
  8. Close the user preferences window

If done correctly, you will now see "SE Anima Sequence (.cs)" under Files -> Export

Notes before animating

Only selected meshes are exported! Everything else like cameras, lights, dummies and unselected objects are ignored.

Make sure your object have a scale of 1.0 before starting animating or the object will be scaled incorrectly on export! To do this go to Object -> Apply or press Ctrl+A and select "Scale" in the list.

The object you animate doesn't actually need to be the model that will be used for the part, you can animate your sequence with any primitive too.

Actions and data paths are irrelevant. Tranformations are extracted from the object per keyframe in timeline, WYSIWYG.

You can't animate 'Hide', it's ignored. As workaround you can set scale to 0.0 and Interpolation Mode to Constant.

If the part will be parent of another part then this must be reflected on Blender too.

You aren't forced to export the parent if the childs are exported, this is for the case you have the same parent motion already exported in another sequence.

The objects names are important for the sequences names if you export multiple objects, on export there's a field called "Class name", the character @ there will be replaced by the object name when an object is processed.

How to export

  1. Select the object(s) you wish to export.
  2. Go to Files -> Export and select "SE Anima Sequence (.cs)" from the list.
  3. Modify export parameters on your bottom-left.
  4. Rename class name for a name of your liking, make sure to use an unique name.
  5. When exporting multiple objects you MUST use the @ character in name or the file will be incorrectly exported with the same class name for each object.
  6. Modify other settings if they don't match what you need.
  7. Select the script folder inside your mod folder and make sure it has ".cs" extension.
  8. Click "Export", there will be a warning box telling you how many objects were actually exported.
  9. Modify your game-logic script and Adquire your new sequences into your parts.
  10. Be amazed seeing your own mod breathing life! :)
